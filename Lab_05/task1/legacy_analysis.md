Проблемы кода 
- глоабльная область видимости: переменные taxRate и currency объвлены через var и достпуны везде, что может вызвать конфликты
- использование var: это конструкция не имеет блочной обалсти видимости и считается устраевшей
- функции конструкторы: function Product и Cart использвуются для создания объектов вместо современных классов
- отсутствие инкапсуляции: все свойства например this.items публичны и могут быть изменены извне публично
- конкатенация строк: используется вместо шаблонныз литералов

разделение на модули:
1. config.mjs - хранятся настрйоки
2. utils.mjs - хранятся вспомогательные функции
3. core.mjs - создаются классы 
4. main.mjs - точка входа где все собирается вместе
