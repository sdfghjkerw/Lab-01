1. решение проблемы global namespace pollution и Lack of structure 
решение: применение namespace pattern . вместо объявления множества глобальных переменных и функций, создат один глобальный объект-контейнер для всего приложения.
результат: все переменные и нукции будут вложены в этот объект что устранит конфликты имен и организует код
2. решение проблемы Tight Coupling
решение: применение module pattern. инкапсулировать функции и переменные связанные внутри IIFE
результат: это создаст закрытую облать видимости , где переменные станут приватными и не удут доступны снаружи и наружу будет экспортироваться только необходимый публичный API
3. code duplicaption
решение : перенести общие фнкции типа валидации email или форматирование цен в тдельные утилитарные модули внутри пространтсва имен
результат: одно место реализации логики позволяет легко изменять ее во всем приложении 
4. проблема с eval() и document.write в таймерах и dom
решение: использовать современные методы и функции. для таймеров - передавать функции напрямую в setTimeout вместо строк. в dom - использовать document.createElement() и appendChild() вместо dicument.write для безопасного манипулирования контентом
