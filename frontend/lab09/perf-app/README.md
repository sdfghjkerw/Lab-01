Данная часть лабораторной работы посвящена методам мемоизации в React для предотвращения избыточных перерисовок (re-renders) и кеширования тяжелых вычислений.

Optimization Techniques Used
1. React.memo
Что делает: Компонент высшего порядка (HOC), который сохраняет результат рендера компонента
Когда использовать: Для "чистых" компонентов, которые часто перерендериваются с одними и теми же пропсами.
Применение в лабе: Компонент UserCard обернут в memo, поэтому он не обновляется при изменении счетчика count в родительском Dashboard.

2. useMemo
Что делает: Хук, который кеширует результат выполнения функции.
Когда использовать: Для дорогих вычислений (циклы, сложные трансформации данных), чтобы не выполнять их при каждом рендере.
Применение в лабе: В AnalyticsChart функция calculateAnalytics запускается только при изменении массива items.

3. useCallback
Что делает: Хук, который кеширует саму ссылку на функцию.
Когда использовать: Когда функция передается как пропс в дочерний компонент, обернутый в memo. Без этого ссылка на функцию будет новой при каждом рендере, что сломает оптимизацию memo.
Применение в лабе: Функция handleIncrement обернута в useCallback, чтобы кнопка Button не перерисовывалась зря.
