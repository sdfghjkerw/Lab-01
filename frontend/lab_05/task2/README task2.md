## Основные концепции TypeScript

### 1. Почему React.ChangeEvent<HTMLInputElement> вместо any?

В TypeScript использование any отключает проверку типов, что сводит на нет все преимущества языка. Использование специфических типов для событий дает следующие преимущества:

* Безопасность типов: Мы гарантируем, что обрабатываем именно событие изменения (ChangeEvent), происходящее на элементе ввода (HTMLInputElement).
* Автодополнение (IntelliSense): IDE подсказывает нам доступные свойства, такие как event.target.value, исключая опечатки.
* Читаемость: Глядя на сигнатуру функции, сразу понятно, какой элемент вызывает это событие.

Пример:

```typescript
// ПРАВИЛЬНО
const handleChange = (event: React.ChangeEvent<HTMLInputElement>) => {
    setSearchTerm(event.target.value);
};

// НЕПРАВИЛЬНО (теряем подсказки и безопасность)
const handleChange = (event: any) => { ... };

```

### 2. Типизация useState: Generics vs Type Inference

#### Type Inference (Вывод типов)

TypeScript умеет автоматически определять тип переменной, основываясь на начальном значении. Если вы инициализируете стейт простым значением, генерики не нужны.

```typescript
const [count, setCount] = useState(0); // TS понимает, что count: number

```

#### Explicit Generics (Явные генерики)

Явное указание типа (useState<T>) необходимо, когда:

1. Начальное значение null или undefined: TypeScript не знает, какой тип будет позже.
2. Сложные структуры данных: При работе с массивами объектов (User[]), явное указание типа гарантирует, что в список попадут только объекты соответствующей структуры.

Пример использования генериков:

```typescript
// Явно указываем, что в стейте будет массив объектов User
const [users, setUsers] = useState<User[]>([]);

```
