pattern name - module pattern
descriprion - инкапсуляция, ипользование замыканий, iife, именование приватных и общедоступных функция
context outline - для упрощение работы кода 
Problem Statement - глобальное засорение переменными, отстсвие приватности, 
solution - код оборачивается в iife, возвращается объект с публичными методами
design - структура - привтаные переменные внутри, публичные показывать всем
examples - базовый module pattern, module pattern с приватными переменными, современный ЕС6 млдуль 
Consequences - из плюсов инкаплсуляция, читсый код, замыкание функций. минусы - сложность тестирование и устаревание ес6
Corequisites - Singleton Pattern, для проверки существования модуля в 1 экземпляре. factory pattern - если модуль должен создавать разные объекты , то внутри него удобно использовать фабрику.
observer pattern - если модуль управляет событиями и должен уведомлять другие части системы, то этот помогает подписывать и оповещать
Relations - Revealing Module Pattern это улучшенная версия module pattern. отличие: все публичные методы явно перечисляются в конце и «раскрываются» наружу. Namespace Pattern - более простой предшественник. создаётся один глобальный объект, внутри которого хранятся все функции
Known Usage - jQuery плагины для расширения функционала. - AngularJS сервисы и контроллеры. - Node.js (CommonJS система модулей).
