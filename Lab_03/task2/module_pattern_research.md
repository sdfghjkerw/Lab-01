исторический контент
в ранних годах javascript не было острой необходимости организации кода из за простых скриптов. дальше появлялись большие веб-приложения с большим количетсвом кода и появилась проблема из за большого количества фнункция/переменных которые засоряли глобальное пространство имен. разработчики придумали module pattern - осоновная задача обернуть код в функцию чтобы создать локальную облатсь видимости а лишние детали скрыть.позже появились стандартизированные сисетым модулей nodejs commonjs amd/requirejs. сейчас в ес6 появились встроенные import/export и стали уже частью языков и заменили классический module pattern

связь с другими модулями
revealing module pattern показывает какие метожы публичные для лучшей читаемости кода. 
namespace pattern - связывает логически идентифиаторы классов функция переменных в уникальные контейнеры, предотвращая конфликты в больших проектх, позволяя давать одинаковые названия в разных контекстах
es6 - современная альтернатива эим двум, но решает задачи удонее и безопасней

использовние в фреймворках и библиотеках
jquerrs использовал Module Pattern для организации плагинов и утилит. angularJS использовал модули для инкапсуляции сервисов и контроллеров. nodeJS до ЕС6 работал на commonjs с require и module.exports
react - до ЕС6 использовался umd/amd для сборки компонентв

современные адаптации 
webpack, babel, vite - современные сборщики используют ЕС6 как основу. typescript - добавил строгую типизацию для модулей
