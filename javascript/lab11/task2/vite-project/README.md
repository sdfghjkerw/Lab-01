# lab 11

## task1

### promise http client
используется для выполнения http запросов через fetch с обработкой ошибок

### retry pattern
повторяет запросы при ошибках с увеличением задержки (exponential backoff)

### async queue
ограничивает количество одновременно выполняемых задач

---

## task2

### parallel fetching
используется promise.all для параллельной загрузки данных

### allsettled
позволяет обработать частичные ошибки без падения всей операции

### async pipeline
данные проходят через цепочку асинхронных функций

### cancellation
используется abortcontroller для отмены запросов

---

## when to use

- promise.all → когда все запросы должны выполниться
- promise.allsettled → когда важны даже частичные результаты
- retry → нестабильная сеть
- queue → ограниченные ресурсы
- pipeline → обработка данных по этапам
- cancellation → пользователь может отменить действие
