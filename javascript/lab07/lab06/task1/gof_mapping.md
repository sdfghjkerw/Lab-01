//////////snippet_02 - observer/////////////

object
объяснение - observer в таблице относится к behavioral. подписчики и издатель взаимодействуют через ссылки на объекты. Поведение реализуется через композицию объектов, а не через иерархию классов

//////////snippet_03 - facade/////////////

object
объяснение - facade в gof таблице - structural. он строится на композиции: один объект (фасад) содержит ссылки на другие объекты подсистемы и делегирует им вызовы

//////////snippet_04 - factory method/////////////
class

//////////snippet_05 - decorator/////////////

object 
объяснение - decorator - structural. он работает через оборачивание одного объекта другим используя композицию а не наследование

//////////snippet_06 - prototype/////////////

object
