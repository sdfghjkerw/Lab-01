
## Обзор проекта
Данная работа посвящена реализации механизмов взаимодействия между объектами в JavaScript с использованием паттернов, минимизирующих прямые зависимости (декаплинг). Работа разделена на две части: система уведомлений (Observer/Pub-Sub) и автоматизация умного дома (Mediator/Command).

---

## Часть 1: Система уведомлений (Observer & Pub/Sub)

### Сравнение паттернов
* **Observer (Наблюдатель):** Создает прямую связь "один-ко-многим". Субъект хранит список наблюдателей и уведомляет их напрямую.
* **Pub/Sub (Издатель-Подписчик):** Вводит посредника (**EventBus**). Издатель не знает, кто на него подписан, а подписчик не знает, кто отправил сообщение. Это обеспечивает максимальную гибкость.

### Управление памятью
Для предотвращения утечек памяти реализован механизм **Cleanup**:
1. Метод `subscribe` возвращает функцию для отписки.
2. Подписчики (Email, Push, Widget) вызывают эти функции при уничтожении, чтобы `EventBus` не хранил ссылки на неиспользуемые объекты.
3. Реализован метод `subscribeOnce`, который автоматически удаляет обработчик после первого срабатывания.

## Часть 2: Умный дом (Mediator & Command)

### Mediator vs Observer
* **Mediator (Медиатор):** Используется для координации сложной логики между группой равноправных объектов (многие-ко-многим). В нашем случае `HomeMediator` управляет связью между светом, термостатом и охраной.
* **Observer:** Больше подходит для простых уведомлений о событии (один-ко-многим).

### Паттерн Command с Undo/Redo
Каждое действие (например, включение света) инкапсулировано в объект команды.
* **Execute:** Выполняет действие.
* **Undo:** Выполняет обратное действие (отмену).
`CommandManager` хранит историю выполненных команд в стеке, что позволяет пользователю отменять действия в обратном порядке.

## Инструкция по запуску

### Вариант 1 (Vite/React)
Если проект запущен через Vite:
1. Установите зависимости: `npm install`
2. Запустите сервер: `npm run dev`
3. Откройте локальную ссылку из терминала.

### Вариант 2 (Обычный HTML)
1. Откройте файл `index.html` через расширение VS Code **Live Server**.
2. Или используйте команду `npx serve .` в корне папки.
