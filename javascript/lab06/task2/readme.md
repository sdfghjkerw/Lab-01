lab 6.2 — pattern selection and minimal implementation

в этой лабораторной работе я анализировала шесть сценариев проектирования и подбирала для каждого подходящее семейство и конкретный паттерн на основе определений из chapter 6.

для task 1 я определила семейство и конкретный паттерн для всех шести сценариев, используя формулировки из gof table и определения семейств: creational, structural и behavioral.

для task 2 я реализовала два сценария:

scenario_01 — singleton
реализовано глобальное конфигурационное хранилище с единственным экземпляром и статическим методом getinstance. повторные вызовы возвращают тот же объект.

scenario_05 — command
каждое действие пользователя представлено отдельным объектом команды с методами execute и undo. класс actionhistory выполняет команды, хранит их в истории и поддерживает undo/redo.

все реализации написаны на es6, используют class, const и современный синтаксис. код запускается через node и содержит демонстрацию работы паттерна через console.log.

основная цель работы — научиться правильно определять категорию паттерна и выбирать архитектурное решение в зависимости от типа проблемы
